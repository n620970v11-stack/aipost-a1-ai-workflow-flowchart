# 保險理賠申請流程

# 流程目標
使要保人於發生事故後，能透過線上提出理賠申請，經公司審核後完成給付或核駁通知，並符合法規給付時效要求。

# 流程範圍
- 涵蓋：從受益人提出理賠申請（含收件）開始，到理賠核准／核駁並完成給付或核駁通知為止
- 不涵蓋：保單契約撤銷（契撤期）、保全變更、理賠爭議申訴／訴訟流程（視為後續流程）
- 繪製方式：各子流程可單獨繪製，亦支援完整端到端流程圖

# 參與角色
- 要保人／受益人
- 客服人員
- 理賠審核人員
- 醫療顧問（視案件複雜度啟動）

# 涉及系統
- 主系統：理賠作業系統
- 外部系統：醫院診斷書驗證、健保資料介接
- 串接系統：核心保單系統（查詢保單／保障內容）、給付系統（撥款作業）

# 子流程清單
1. 理賠申請受理
2. 文件審核與資料蒐集（待補）
3. 理賠核決（含醫療顧問複核）（待補）
4. 理賠給付與結案通知（待補）

> 目前僅完成「子流程 1：理賠申請受理」的詳細內容，其餘子流程細節後續再滾動補充。

# 子流程 1：理賠申請受理

## 目標
確認要保人身份及保單有效性，完整收件並建立理賠案件。

## 觸發條件
要保人透過線上提出理賠申請。

## 前置條件
- 保單為有效契約
- 保險事故已發生

## 涉及角色
- 要保人
- 客服人員

## 涉及系統
- 主系統：理賠作業系統
- 串接系統：核心保單系統（查詢保單狀態與保障內容）

## 輸入
- 理賠申請書
- 身份證明文件
- 診斷證明書

## 流程步驟
1. 受益人提交理賠申請與相關文件（線上上傳或臨櫃遞交）
2. 客服人員核對申請人身分
3. 系統調用核心保單系統查詢保單狀態與保障內容（回傳：保單有效性、保障項目）
4. 系統建立理賠案件並產生案件編號

## 判斷點
- 保單是否有效
  - 是：進入文件審核與資料蒐集
  - 否：退件並通知申請人保單狀態異常

## 成功結果
- 理賠案件成功建立，案件編號產生，進入下一階段審核

## 失敗結果
- 保單無效或申請文件不齊全，退件通知申請人補件或說明原因

## 輸出
- 理賠案件編號
- 受理清單（含案件基本資料）

## 下一步
- 子流程 2：文件審核與資料蒐集
