# 流程名稱
設備報修流程

# 流程目標
說明使用者進入 IT 即時語音平台後，到完成設備報修單的完整流程，作為後續流程圖生成依據。

# 流程範圍
- 涵蓋：從使用者進入 IT 即時語音平台，到完成設備報修單建立的端到端流程
- 繪製方式：各子流程可單獨繪製，也支援完整端到端流程圖

# 參與角色
- 使用者
- AI 語音助理

# 涉及系統
- IT 即時語音平台
- AI 語音助理（GPT Realtime 模型）
- MCP Server
- 設備報修系統

# 子流程清單
1. 進入即時語音平台
2. 申報人身分資訊確認
3. 申報人名下設備顯示與選擇
4. 設備問題描述與確認
5. 建立設備報修單

# 子流程 1：進入即時語音平台

## 目標
確認使用者可正常進入即時語音平台並具備可用互動能力。

## 觸發條件
使用者進入即時語音平台頁面。

## 前置條件
- 使用者具備即時語音平台使用權限

## 涉及角色
- 使用者
- AI 語音助理

## 涉及系統
- IT 即時語音平台

## 輸入
- 使用者進入平台頁面的動作

## 流程步驟
1. 使用者進入「資訊服務即時語音平台」頁面
2. 平台檢查網頁是否已開啟麥克風權限
3. 平台檢查是否已成功建立即時連線
4. 平台依麥克風權限與連線狀態，顯示可用的互動方式

## 判斷點
- 網頁是否已開啟麥克風權限
  - 否：
    - 麥克風呈現禁用狀態
    - 使用者可改以文字輸入與 AI 語音助理互動
  - 是：麥克風呈現啟用狀態，使用者可使用語音或文字互動
- 即時連線是否建立成功
  - 是：進入可互動的即時語音頁面
  - 否：
    - 平台顯示連線失敗訊息
    - 使用者點擊彈窗確認
    - 平台重新載入頁面並再次嘗試建立連線
    - 再次確認連線是否成功
      - 是：進入即時語音頁面
      - 否：繼續顯示連線失敗訊息，使用者稍後再試

## 成功結果
- 使用者成功進入可互動的即時語音頁面

## 失敗結果
- 連線失敗，使用者目前無法使用平台並需稍後再試

## 輸出
- 平台頁面可用狀態
- 麥克風與文字輸入可用狀態

## 下一步
- 申報人身分資訊確認

# 子流程 2：申報人身分資訊確認

## 目標
讓 AI 語音助理辨識使用者的設備報修意圖，並完成申報人身分資訊確認。

## 觸發條件
使用者已進入資訊服務即時語音平台，並準備進行設備報修。

## 前置條件
- 使用者已進入即時語音平台頁面

## 涉及角色
- 使用者
- AI 語音助理

## 涉及系統
- 資訊服務即時語音平台
- 設備報修系統
- MCP Server

## 輸入
- 使用者語音輸入
- 使用者文字輸入

## 流程步驟
1. 使用者以語音或文字輸入「我要進行設備報修」
2. AI 語音助理接收輸入並辨識報修意圖
3. AI 語音助理在辨識成功後，透過 MCP Server 呼叫設備報修系統 API
4. 設備報修系統回傳使用者身分資訊
5. AI 語音助理以文字顯示申報人身分資訊
6. 使用者確認身分資訊是否正確

## 判斷點
- AI 語音助理是否成功辨識報修意圖
  - 否：請使用者重新說明需求，返回意圖辨識步驟
  - 是：查詢申報人身分資訊
- 設備報修系統是否成功回傳身分資訊
  - 否：顯示查詢失敗訊息，請使用者稍後再試或聯繫內部客服專線，流程結束
  - 是：顯示申報人身分資訊
- 使用者身分資訊是否正確
  - 否：請使用者聯繫內部客服專線，流程結束
  - 是：進入「申報人名下設備顯示與選擇」流程

## 成功結果
- 系統成功辨識報修意圖
- 使用者身分資訊確認正確

## 失敗結果
- 意圖辨識失敗，需重新輸入
- 身分資訊查詢失敗或資訊不正確，需改由內部客服專線協助

## 輸出
- 已確認的申報人身分資訊

## 下一步
- 申報人名下設備顯示與選擇

# 子流程 3：申報人名下設備顯示與選擇

## 目標
取得申報人名下的設備資訊，並讓使用者選擇本次欲報修的設備。

## 觸發條件
使用者已完成身分資訊確認。

## 前置條件
- 申報人身分資訊已確認正確

## 涉及角色
- 使用者
- AI 語音助理

## 涉及系統
- 資訊服務即時語音平台
- 設備報修系統
- MCP Server

## 輸入
- 已確認的申報人身分資訊

## 流程步驟
1. AI 語音助理透過 MCP Server 呼叫設備報修系統 API
2. 設備報修系統依申報人身分查詢名下設備
3. 設備報修系統回傳設備清單
4. AI 語音助理以文字顯示可報修的設備資訊
5. 使用者選擇欲報修的設備
6. AI 語音助理顯示使用者選擇的設備，請使用者確認

## 判斷點
- 設備報修系統是否成功回傳設備清單
  - 否：顯示查詢失敗訊息，請使用者稍後再試或聯繫內部客服專線，流程結束
  - 是：顯示設備清單
- 申報人名下是否有可報修設備
  - 否：告知使用者目前無可報修設備，請聯繫內部客服專線，流程結束
  - 是：請使用者選擇欲報修的設備
- 使用者確認選擇的設備是否正確
  - 否：返回設備清單，請使用者重新選擇
  - 是：進入「設備問題描述與確認」流程

## 成功結果
- 已取得申報人名下的設備清單
- 已確認本次欲報修的設備

## 失敗結果
- 設備清單查詢失敗，或申報人名下沒有可報修設備

## 輸出
- 已確認的報修設備資訊

## 下一步
- 設備問題描述與確認

# 子流程 4：設備問題描述與確認

## 目標
引導使用者說明設備問題，並在建立報修單前確認問題描述正確。

## 觸發條件
使用者已確認本次欲報修的設備。

## 前置條件
- 已取得報修設備資訊

## 涉及角色
- 使用者
- AI 語音助理

## 涉及系統
- 資訊服務即時語音平台

## 輸入
- 報修設備資訊
- 使用者的語音或文字問題描述

## 流程步驟
1. AI 語音助理詢問使用者「設備遇到什麼問題？」
2. 使用者以語音或文字說明設備問題
3. AI 語音助理將辨識到的問題描述顯示於頁面
4. AI 語音助理詢問使用者問題描述是否正確
5. 使用者確認問題描述，或重新說明問題

## 判斷點
- AI 語音助理是否成功辨識問題描述
  - 否：請使用者重新說明問題，返回問題輸入步驟
  - 是：顯示問題描述並請使用者確認
- 使用者確認問題描述是否正確
  - 否：使用者重新說明問題，返回問題輸入步驟
  - 是：進入「建立設備報修單」流程

## 成功結果
- 已完成設備問題描述與確認

## 失敗結果
- 問題描述無法辨識，需請使用者重新輸入

## 輸出
- 已確認的設備問題描述

## 下一步
- 建立設備報修單

# 子流程 5：建立設備報修單

## 目標
將已確認的申報人、設備與問題描述送至設備報修系統，完成設備報修單建立並回傳結果。

## 觸發條件
使用者已確認設備問題描述。

## 前置條件
- 已確認申報人身分資訊
- 已確認報修設備資訊
- 已確認設備問題描述

## 涉及角色
- 使用者
- AI 語音助理

## 涉及系統
- 資訊服務即時語音平台
- 設備報修系統
- MCP Server

## 輸入
- 已確認的申報人身分資訊
- 已確認的報修設備資訊
- 已確認的設備問題描述

## 流程步驟
1. AI 語音助理整理報修所需資訊
2. AI 語音助理透過 MCP Server 呼叫設備報修系統 API
3. MCP Server 將申報人、設備與問題描述送至設備報修系統
4. 設備報修系統進行報修單建立並回傳處理結果
5. AI 語音助理依回傳結果顯示報修成功或失敗訊息
6. 報修成功時，使用者可透過前台畫面點擊查看報修內容

## 判斷點
- 設備報修單是否建立成功
  - 是：顯示報修成功訊息與報修細節查詢入口，流程結束
  - 否：顯示報修失敗訊息，請使用者稍後再試或聯繫內部客服專線，流程結束

## 成功結果
- 設備報修單建立成功
- 使用者可查看報修細節

## 失敗結果
- 設備報修單建立失敗，使用者需稍後再試或改由內部客服專線協助

## 輸出
- 報修結果
- 報修細節查詢入口（建立成功時）

## 下一步
- 流程結束
